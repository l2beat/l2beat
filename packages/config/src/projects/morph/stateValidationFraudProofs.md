Morph uses a one round fault proof system where whitelisted Challengers, if they find a faulty state root within the {{challengeWindow}} challenge window,           can post a {{challengeBond}} WEI bond and request a ZK proof of the state transition. At least 5 Challengers are operated by entities external to the team. After the challenge, during a {{proofWindow}} proving window, a ZK proof must be           delivered, otherwise the state root is considered invalid and the root proposer bond, which is currently set to {{stakingValue}} ETH, is slashed. The zkVM used is SP1 by Succinct.          If a valid proof is delivered, the Challenger loses the challenge bond. The Morph Multisig can revert unfinalized batches.
