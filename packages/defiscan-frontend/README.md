# DeFiScan Frontend

Standalone public website that renders DeFi protocol security reviews from pre-compiled JSON data. Built with React, Vite, TailwindCSS, and Recharts.

## Quick Start

```bash
# From the repo root
cd packages/defiscan-frontend

# Development server (http://localhost:5173)
pnpm dev

# Production build (runs compile-data first, outputs to build/)
pnpm build

# Type checking
pnpm typecheck
```

## Architecture

### Data Model

The frontend is **fully static** — it reads pre-compiled JSON files at build time and runtime, with no backend API calls.

```
public/data/
├── index.json                          # Aggregated protocol list + global stats
├── liquity-v2/
│   ├── compiled-review.json            # Full review data (from ReviewCompiler)
│   └── funds-data.json                 # Optional: live funds data for patching
└── Steakhouse-USDC/
    ├── compiled-review.json
    └── funds-data.json
```

- **`compiled-review.json`**: Generated by `ReviewCompiler` (in `packages/l2b/`). Contains protocol metadata, admins, dependencies, funds, functions, contracts, and sections.
- **`funds-data.json`**: Optional. If present alongside a compiled review, the `compile-data` script patches fund balances/positions into the review at build time.
- **`index.json`**: Generated by `scripts/compile-data.ts` at build time. Aggregates protocol summaries, global totals, and cross-protocol dependency data.

### Build Pipeline

```
[ReviewCompiler]  →  compiled-review.json  →  [compile-data.ts]  →  index.json
   (l2b / monitor)     (per project)            (build time)         (aggregated)
```

1. **ReviewCompiler** (in `packages/l2b/`) compiles all project data into `compiled-review.json`
2. **compile-data.ts** runs as a prebuild step, scanning `public/data/` for compiled reviews
3. It patches funds data, aggregates protocol summaries, and writes `index.json`

### Adding a New Protocol

1. Run the review workflow in protocolbeat (permissions, call graph, review config)
2. Click "Compile Review" in the Terminal panel — output is written directly to `public/data/<slug>/compiled-review.json`
3. Optionally copy `funds-data.json` alongside it
4. Run `pnpm build` or `pnpm dev` — the protocol appears automatically

## Pages

### Landing Page (`/`)

Protocol listing table with sortable columns, global stats (total capital reviewed, protocols count), and top cross-protocol dependencies.

### Review Page (`/protocol/:slug`)

Three view modes toggled via tabs:

- **Report View**: Summary, key findings, fund holders ("What Is at Stake?"), admin cards ("Who Is in Control?"), dependency cards, and "More Information" section with code/audits. Sticky section navigation.
- **Explorer View**: Tabbed data explorer with Contracts, Functions, Admins, Dependencies, Funds tabs. Includes SVG diagrams (capital flow, dependency risk, direct vs reachable).
- **Dashboard View**: Visual dashboard with exposure charts (treemaps, pie charts) and protocol statistics.

### About Page (`/about`)

Marketing-style page with a hero ("The Transparency Layer for DeFi."), Our Mission section with live stats (protocol count and total TVL from `useIndex()`), an "Our Approach" three-card grid (Open Source Logic → GitHub repo, Verifiable Data → Lens, Protocol Mapping → docs), and a "Built and Backed by DeFi Veterans" section with a CTA linking to The DeFi Collective.

### Compare Page (`/compare`)

Side-by-side protocol comparison:

- Protocol selector table with checkboxes
- Radar charts, scatter plots, admin type distribution
- Shared dependency map, rankings panel, benchmark bars
- URL state: `?protocols=slug1,slug2` for shareable comparisons

## Key Types

The frontend types in `src/types.ts` mirror the backend's `CompiledReview` interfaces from `packages/l2b/.../reviewCompiler.ts`. Key types:

- `CompiledReview` — full review data structure
- `CompiledAdmin` — admin with type, functions, capital at risk
- `CompiledDependency` — external dependency with called functions
- `CompiledFundHolder` — contract holding funds (balances + positions + token info)
- `CompiledFunction` — permissioned function entry
- `CompiledContract` — contract with tags (external, governance, entity)
- `ProtocolSummary` — lightweight summary for the landing page index
